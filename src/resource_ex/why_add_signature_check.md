# 有关 「id 校验」 的说明（资源包创作者请注意）

## 为什么要添加 id 签名校验？

MetaMystia 为保证资源包的整型 id 之间不相冲突，设计了白名单的签名校验。目的是在高度自由的前提下保证每个包 id 的全局唯一性。**MetaMystia 不会限制每个资源包创作者的合理申请，但给出签名也并不代表对非法资源包进行支持推荐**。

## id 段划分

计划将整个 `int` 可用范围划分为三段

- `-2147483648<=?<= 8999` 段，为游戏保留
- `9000<=?<=1073741823` 为受管理区，需要向 MetaMystia 申请后使用
- `1073741823<=?<=2147483647` 为自由区，无需申请即可使用

## 如何申请签名？

向 MetaMystia 以 邮箱(`MetaMiku@hotmail.com`)、qq 群(1034953242) 或本仓库 issue 中提供自己所创作的资源包 label (如 `ResourceExample`) 以及 「所申请的 id 段」（如 `23000~23999`）即可获取*签名*，MetaMystia 保证该 id 段不会重复分配给其他资源包，进而保证资源包之间的 id 不会冲突。申请后，**除非扩展 id 段或修改资源包 label，无需进行重复签名**。

## 暂时没有签名或者不想申请签名怎么办？

资源包创作者亦可修改 `Touhou Mystia Izakaya/BepInEx/config/MetaMystia.cfg` 中的 `SignatureCheck` 项来禁止自己的 MetaMystia.dll 进行签名检查，用以在申请到签名前进行临时性的创作调试。

如有资源包创作者对「白名单签名」制不满意，可以使用自由区 id 段，使用该段 id 的资源包不会被任何 MetaMystia.dll 进行签名检查，但需要您自行控制 id 冲突的隐患与可能。
